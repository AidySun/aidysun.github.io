hugo 主题本地没问题，github.io中不起作用

Failed to find a valid digest in the 'integrity' attribute for resource 'https://.....css' with computed SHA-256 integrity '.....'. The resource has been blocked.
March 28, 2023 · SZH

使用 PaperMode 主题，本地渲染没问题，将 publish 文件夹 push 到 <username>.github.io 后，通过 Github Page 访问会出现 CSS block 的情况：

尝试过但不起作用的方法#

  1. 修改 config.toml 中的 baseURL，用 http、用 https。
  2. 根据 head, footer: add option to disable fingerprinting adityatelange#89, 在 config.toml 中加入如下内容:
1[params.assets]
2  disableFingerprinting = true

我的解决方法#

根据 Failed to find a valid digest in the integrity attribute，问题可能是 git line breaking 的问题，我使用的是 macOS，当时的 git config -l 中:

1core.autocrlf=true

在 hugo project repo 和 我的 github.io repo 中 执行 git config core.autocrlf false。

然后生成博客：

1cd blog-hugo
2hugo -t PaperMod
3
4# submodule publish is <username>.github.io
5cd publish
6git add .
7git commit -m "update"
8git push

其它未尝试的方法#

  • 修改模版中的 integrity="{{ xxxx }}" 为 integrity=""
  • git config --global core.autocrlf input
