Godot提供了几种碰撞对象，既提供碰撞检测，又提供碰撞响应。在决定在项目中使用哪种对象时可能会感到困惑。如果您了解每个对象的工作原理、优缺点，就可以避免问题并简化开发。在本教程中，我们将介绍CharacterBody2D节点，并展示如何使用它的一些示例。

注意

虽然本文档在示例中使用CharacterBody2D，但相同的概念也适用于3D。

什么是角色体（Character Body）？¶ CharacterBody2D用于通过代码控制的角色体。角色体在移动时会检测与其他物体的碰撞，但不受引擎物理属性（如重力或摩擦力）的影响。虽然这意味着您必须编写一些代码来创建它们的行为，但也意味着您对它们的移动和反应有更精确的控制。

注意

本文档假设您已经熟悉Godot的各种物理体。请先阅读物理介绍，以获得物理选项的概述。

提示

CharacterBody2D可以受重力和其他力的影响，但您必须在代码中计算其移动。物理引擎不会移动CharacterBody2D。

移动和碰撞¶ 在移动CharacterBody2D时，不应直接设置其position属性。相反，您可以使用move_and_collide()或move_and_slide()方法。这些方法沿着给定的向量移动角色体，并检测碰撞。

警告

应在_physics_process()回调中处理物理体的移动。

这两种移动方法具有不同的用途，稍后在本教程中，您将看到它们的工作示例。

move_and_collide¶ 该方法接受一个必需的参数：一个表示物体相对移动的Vector2。通常，这是您的速度向量乘以帧时间步长（delta）。如果引擎在这个向量的任何位置检测到碰撞，物体将立即停止移动。如果发生碰撞，该方法将返回一个KinematicCollision2D对象。

KinematicCollision2D是一个包含有关碰撞和碰撞对象的数据的对象。使用这些数据，您可以计算碰撞响应。

当您只想移动物体并检测碰撞，但不需要任何自动碰撞响应时，move_and_collide非常有用。例如，如果您需要一个在墙壁上弹射的子弹，您可以在检测到碰撞时直接改变速度的角度。请参见下面的示例。

move_and_slide¶ move_and_slide()方法旨在简化常见情况下需要一个物体沿另一个物体滑动的碰撞响应。它在平台游戏或顶视图游戏中特别有用，例如。

在调用move_and_slide()时，该函数使用一些节点属性来计算滑动行为。这些属性可以在检查器中找到，或者在代码中设置。

velocity - 默认值：Vector2(0, 0)

此属性表示物体的速度向量，以像素为单位。move_and_slide()将在碰撞时自动修改此值。

motion_mode - 默认值：MOTION_MODE_GROUNDED

该属性通常用于区分侧向滚动和顶视图运动。在使用默认值时，您可以使用is_on_floor()、is_on_wall()和is_on_ceiling()方法来检测物体与哪种类型的表面接触，并且物体将与斜坡交互作用。当使用MOTION_MODE_FLOATING时，所有碰撞将被视为“墙”。

up_direction - 默认值：Vector2(0, -1)

该属性允许您定义引擎应该将哪些表面视为地板。它的值使您可以使用is_on_floor()、is_on_wall()和is_on_ceiling()方法来检测物体与哪种类型的表面接触。默认值意味着水平表面的顶部将被视为“地面”。

floor_stop_on_slope - 默认值：true

此参数防止物体在静止时下滑。

wall_min_slide_angle - 默认值：0.261799（弧度，相当于15度）

这是物体在撞到斜坡时允许滑动的最小角度。

floor_max_angle - 默认值：0.785398（弧度，相当于45度）

此参数是在一个表面不再被视为“地板”之前的最大角度。

还有许多其他属性可用于在特定情况下修改物体的行

检测碰撞¶ 使用move_and_collide()函数时，它会直接返回一个KinematicCollision2D对象，您可以在代码中使用它。

使用move_and_slide()时，可能会发生多个碰撞，因为滑动响应是计算出来的。要处理这些碰撞，请使用get_slide_collision_count()和get_slide_collision()：

使用move_and_collide。¶

var collision = move_and_collide(velocity * delta) if collision: print(“我与”, collision.get_collider().name, “发生碰撞”)

使用move_and_slide。¶

move_and_slide() for i in range(get_slide_collision_count()): var collision = get_slide_collision(i) print(“我与”, collision.get_collider().name, “发生碰撞”) 注意

get_slide_collision_count()只计算物体发生碰撞并改变方向的次数。

有关返回的碰撞数据的详细信息，请参见KinematicCollision2D。

选择使用哪种移动方法？¶ 新手在使用Godot时常常会问：“如何决定使用哪种移动函数？”通常的回答是使用move_and_slide()，因为它看起来更简单，但这并不一定是正确的。可以将move_and_slide()视为一种特殊情况，而move_and_collide()更通用。例如，以下两个代码片段会产生相同的碰撞响应：

using move_and_collide¶

var collision = move_and_collide(velocity * delta) if collision: velocity = velocity.slide(collision.get_normal())

using move_and_slide¶

move_and_slide() 使用move_and_collide()和move_and_slide()可以实现相同的碰撞响应，但可能需要更多的代码。然而，正如下面的示例所示，有些情况下move_and_slide()无法提供所需的响应。

在上面的示例中，move_and_slide()会自动修改速度变量。这是因为当角色体与环境发生碰撞时，函数会重新计算速度以反映减速效果。

例如，如果您的角色掉到地板上，您不希望它因重力效果而积累垂直速度。相反，您希望其垂直速度重置为零。

move_and_slide()还可以在循环中多次重新计算运动物体的速度，以产生平滑的运动。默认情况下，它将移动角色体并最多发生五次碰撞。在该过程结束时，角色体的新速度可在下一帧中使用。

示例¶ 要查看这些示例的实际效果，请下载示例项目：character_body_2d_starter.zip

移动和墙壁¶ 如果您已经下载了示例项目，可以在"basic_movement.tscn"中找到此示例。

对于此示例，请添加一个CharacterBody2D节点，并作为其子节点添加一个Sprite2D和一个CollisionShape2D。使用Godot的"icon.png"作为Sprite2D的纹理（将其从文件系统窗口拖动到Sprite2D的纹理属性中）。在CollisionShape2D的Shape属性中，选择"New RectangleShape2D"并调整矩形的大小以适应精灵图像。

将脚本附加到CharacterBody2D，并添加以下代码：

var speed = 300

func get_input(): var input_dir = Input.get_vector(“ui_left”, “ui_right”, “ui_up”, “ui_down”) velocity = input_dir * speed

平台游戏的移动方式¶ 让我们尝试另一个常见的示例：2D平台游戏。move_and_slide()非常适合快速构建一个功能完整的角色控制器。如果您已经下载了示例项目，您可以在"platformer.tscn"中找到它。

对于此示例，我们假设您有一个由一个或多个StaticBody2D对象构成的关卡。它们可以是任何形状和大小。在示例项目中，我们使用Polygon2D来创建平台的形状。

以下是角色体的代码：

extends CharacterBody2D

var speed = 300.0
var jump_speed = -400.0

# 从项目设置中获取重力，以便与刚体节点同步。
var gravity = ProjectSettings.get_setting("physics/2d/default_gravity")


func _physics_process(delta):
    # 添加重力。
    velocity.y += gravity * delta

    # 处理跳跃。
    if Input.is_action_just_pressed("jump") and is_on_floor():
        velocity.y = jump_speed

    # 获取输入方向。
    var direction = Input.get_axis("ui_left", "ui_right")
    velocity.x = direction * speed

    move_and_slide()
    ```

    在这段代码中，我们使用了如上所述的move_and_slide()来沿着速度向量移动角色体，并在地面或平台等碰撞表面上滑动。我们还使用is_on_floor()来检查是否允许跳跃。如果没有这个检查，您将能够在空中"跳跃"；这对于制作Flappy Bird游戏很好，但对于平台游戏来说不适用。

一个完整的平台游戏角色还有很多其他方面要考虑：加速度、双重跳跃、短暂悬空时间等等。上面的代码只是一个起点。您可以将其作为基础，根据自己的项目需要扩展成所需的移动行为。
