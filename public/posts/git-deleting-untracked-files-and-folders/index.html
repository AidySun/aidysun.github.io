还有一种情况就是已经在 .gitignore 中的文件或文件夹是不会被删除的。

需要通过 -x 选项来删除被 .gitignore 包含的文件或文件夹：

git clean options

Option Comments
-f Delete files or diretories
-d Recurse into untracked directories
-x Don’t use the standard ignore rules

(End)
